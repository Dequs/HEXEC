## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

Do **not** disclose the vulnerability publicly, including through issues, discussions, or social media.

### How to report
- Contact the project maintainer directly or open a **private security report** if the platform supports it
- Provide a clear and detailed description of the vulnerability
- Include steps to reproduce the issue and any relevant logs or proof of concept, if available

### What to expect
- You should receive an initial response within **5 business days**
- If the vulnerability is confirmed, a fix will be developed and released in a future update
- If the report is declined, you will be informed with an explanation

Please allow reasonable time for the issue to be addressed before any disclosure.
Responsible reporting helps keep the project and its users secure.
